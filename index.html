01 · 2026.05 · 项目负责人 / AI 产品设计 / 全栈开发

多模型个人 AI Agent 框架

把多模型调用、分层记忆和工具执行放进同一个个人 AI 工作流，重点解决模型选择、信息沉淀和工具风险确认。

技术栈
TypeScript / Node.js / WebSocket / OpenAI-Compatible API
关键词
多模型路由、短中长期记忆、SSE 流式对话、WebSocket 会话隔离、工具权限确认
用户输入 Chat / Task / Tool Request
决策层 Model Router Memory · Session · Permission
执行层 LLM / SSE / WebSocket / Tools

项目背景

个人高频使用 AI 时，聊天、任务执行和记录沉淀分散在不同模型和工具里，长期上下文容易丢失，高风险操作也缺少确认机制。

我负责什么

定义产品目标和模块边界，设计多模型路由、记忆分层、工具权限确认流程，并完成服务端通信与基础前端。

产品价值

让 AI 从一次性问答变成可持续使用的个人工作流入口，同时把模型选择和工具风险控制放进产品机制里。

02 · 2026.05 · 项目负责人 / AI 产品设计 / 前端开发

Final Exam Review

从复习资料出发生成题目，再用掌握度和错题循环组织练习，让“看资料”变成“持续发现薄弱点并补强”。

技术栈
Next.js 15 / React 19 / sql.js / SQLite / OpenAI-Compatible API
核心能力
多格式上传、AI 自动出题、4 档出题策略、掌握度 0-5 级、错题下一轮

项目背景

期末复习材料来源杂、题目少、薄弱点不明确，普通 AI 问答很难形成持续练习和下一轮复习计划。

我负责什么

设计上传、出题、答题、判定、掌握度更新和错题回流流程，完成前端应用、题库数据结构和 AI 出题策略配置。

产品价值

把静态资料转成可练习、可追踪、可迭代的复习路径，降低学生从资料到题库的转换成本。

03 · 2026.04 · 项目负责人 / 产品设计 / 桌面端开发

模拟输入系统

将一段文本拆成连续键盘输入事件，并按可控节奏输入到目标窗口。重点是输入过程稳定、可观察、可中断。

核心能力
文本拆分、连续键盘输入事件、节奏控制、目标窗口输入、异常中断处理
边界
不写跨屏能力，不写手机控制电脑。

项目背景

部分桌面使用场景需要把现有文本稳定输入到目标窗口，用户需要更细粒度的输入节奏和状态反馈。

我负责什么

定义输入前确认、输入中状态、暂停/中断和异常提示流程，完成桌面端输入事件调度与基础配置界面。

产品价值

把重复手动输入变成可控的工具流程，减少用户操作负担，同时保持功能边界清晰。

04 · 2026.02-04 · 项目负责人 / 产品设计 / 桌面端与后端开发

跨屏输入系统

手机浏览器连接电脑端，将手机文本发送到电脑端接收、展示和复制，解决跨设备内容流转中的摩擦。

核心能力
手机浏览器连接电脑端、文本发送、电脑端接收与展示、复制使用、连接状态处理
边界
不写模拟输入，不写禁止粘贴绕过。

项目背景

手机里的一段文字要转到电脑端继续处理时，经常依赖聊天软件或临时文件，步骤多且不适合轻量、即时的内容流转。

我负责什么

设计手机连接、电脑接收、内容展示、复制使用和连接状态反馈流程，完成桌面端、本地服务和打包交付。

产品价值

把跨设备文本转移从“绕一圈发送给自己”变成短路径，提高日常工具使用效率。

05 · 2026.02-04 · 项目负责人 / 产品设计 / 全栈开发

独立站与软件授权支付系统

面向独立软件产品，从商品配置、下单支付、回调验签、主动查单到激活码发放，搭建完整商业化链路。

技术栈
Next.js / Drizzle ORM / PostgreSQL / Alipay SDK / Ed25519
核心能力
支付宝支付、异步通知验签、主动查单、幂等发放、Ed25519 激活码、后台商品管理

项目背景

独立软件从免费试用走向付费时，需要可靠处理支付、权益发放、激活码校验和后台商品管理，而不是只做一个收款页面。

我负责什么

设计订单、支付、授权、后台管理流程，完成前后端、数据库、支付宝接口封装和 Ed25519 激活码签发校验。

产品价值

补齐工具产品商业化最关键的一段：用户付费后能够稳定获得权益，后台也能管理商品、订单和授权。
